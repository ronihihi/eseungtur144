Security.txt
A05
 
Description
A security.txt file is a standardized text file that provides a clear way for security researchers and users to report vulnerabilities. Located at https://example.com/.well-known/security.txt, it serves as a "contact card" for responsible disclosure, offering details like contact emails, encryption keys, disclosure policy URLs, and acknowledgment programs. By using security.txt, organizations enhance transparency and encourage prompt reporting of vulnerabilities, ensuring reports reach the right team. Adopting this standard shows a commitment to cybersecurity and facilitates collaboration with ethical hackers to protect users.
Recommendation
To improve security, add a security.txt file to your domain for easy vulnerability reporting. Include essential fields like Contact (for reporting issues), and optionally, Encryption (PGP key), Policy (security policies), and Acknowledgements (crediting contributors). Place the file in the .well-known directory, accessible at https://example.com/.well-known/security.txt over HTTPS.

Forced browsing
A01
 
Description
Forced browsing is a security vulnerability where attackers guess URLs or use automated tools to access restricted files, exposing sensitive data like configurations, admin panels, and backups if access controls are weak.
Recommendation
Prevent forced browsing with strict access controls, 403 Forbidden responses, and hidden sensitive files. Use URL whitelisting, web application firewall, and regular security audits to block attacks.

.htaccess is publicy readable
A05
 
Description
The .htaccess file defines critical server configurations like access controls, redirects, and security headers. If exposed, attackers can exploit it to uncover hidden directories, authentication rules, or server settings, leading to potential security risks.
Recommendation
To prevent unauthorized access, configure the server to deny public access to .htaccess files by adding the following rule in the main Apache configuration or another .htaccess file: <Files ".htaccess"> Require all denied </Files> Additionally, set file permissions to 600 (chmod 600 .htaccess) to ensure only the owner can read or modify it.
